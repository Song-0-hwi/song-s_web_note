웹 클라이언트와 웹 서버의 관계

한 대의 컴퓨터에 설치된 Web Browser가 접속하려는 주소에 해당되는 서버 컴퓨터(웹 서버가 설치되어 있는)에 인터넷을 통해 전기적 신호를 보낸다.
그럼 서버 컴퓨터가 해당되는 디렉토리 안의 파일을 또다시 전기적 신호로 보내면, 웹 브라우저가 그 파일의 코드를 해석해서 우리에게 보여준다
즉, 웹 서버는 내 컴퓨터에 있는 문서를 전세계의 누구나 언터넷이 연결되어 있는 컴퓨터에 웹브라우저를 깔면 가져다가 볼 수 있도록 한다.
여기서 아마 hacking이 생겨나지 않았나... 혜림아 기대해.

인터넷에 연결된 컴퓨터 하나 하나를 호스트. 이런 컴퓨터를 빌려주는 사업을 호스팅.
난 코딩을 해서 github에 올려주면 그 코드를 한 서버에 depoly해준다. github는 hosting을 해줘 주소를 부여하고, 서버를 활성화

웹서버 운영은 컴퓨터에 웹서버를 설치해서 다른 컴퓨터가 통신규약(protocol)에 따라, 네트워크를 통해 IP주소로 다른 컴퓨터가 접근할 수 있도록 하는 것이다. 한대의 컴퓨터를 초월해서 전 서계에 있는 수많은 컴퓨터를 마치 내 컴퓨터인 것처럼 사용할 수 있게 된 웅장한 진보라 할 수 있지만, 악용될 수도 있단 생각이 든다.

태그의 슈퍼스타

문서의 기초는 구조이다. 따라서 우리는 이 약속을 지키며 문서를 작성해야한다.
우리가 작성한 파일은 utf-8이라는 방식으로 저장된다. 따라서 열때도 utf-8의 방식으로 해야한다. 이 부분은 우리가 페이지의 제목 아래 파일들을 저장하는 방식을 표시한 것이다. 제목과 meta는 이 문서를 설명하는 부분이므로 head라는 태그로 묶어준다.
또한 본문은 body라는 태그로 묶어주고, body 태그와 head 태그를 감싸는 하나의 태그를 두기로 약속하는데 그게 html 태그이다.
마지막으로 이 웹페이지가 HTML로서 만들어졌다는 것을 표현하기 위해서 문서의 시작에 !doctype html이라는 코드를 추가한다.

웹서버 운영

웹서버와 웹브라우저가 상호작용하여 html파일들을 불러온다. 하지만 자신의 컴퓨터안 파일을 열 경우, 웹 브라우저만 개입할 수 있다. 반면에 처음 말했듯이 웹브라우저와 웹서버, 둘다 개입하는 경우 http라는 서로 통신할 때 사용하는 통신 규약을 통해 웹페이지를 주고받는다.
